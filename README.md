# homelab

Container orchestration for homelab.

## What is this?

A collection of `compose.yaml`, `Dockerfile`, and configs for the self-hosted services I use for my homelab (or public server).

## Why do I make this?

I have a public server that hosts my personal cloud services. For a long time, the services have been deployed on bare metal, and they worked for years without any issue.
However, things become challenging when I want to upgrade some server components due to security vulnerabilities, and it becomes even more complicated if I want to migrate the services to a new server.
After a couple of breaking changes and dealing with multiple legacy configs, I've decided to adopt IaC (Infrastructure as Code) to provide a declarative and version-controlled infrastructure for my personal services.
I hope this will reduce my future mental burden of service management, and might be helpful to people under the same situation.
